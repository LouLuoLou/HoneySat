Main Functionalities

Why this project?

The long-term goal is to replace the current telnet and remote desktop setup with direct communication to the zeroMQ hub of the ground station.

Web Interface for {{ mission_name }} Missions

  • Communicate with {{ mission_name }} Missions: The web interface allows you to interact with and receive telemetry from the {{ mission_name }} missions.
  • Remote Desktop: Access a pre-configured machine via a web version of VNC, designed to be more reliable than previous setups.
  • Documentation: Access documentation related to the interface and its functionalities.

Scheduled Features

Current Features

  • Satellite Position Map: View the current position of the selected satellite on a map (implemented).
  • Next Pass Information: See information about the next satellite pass (implemented).
  • Telnet Terminal:
    • Display the last output from the telnet terminal (implemented).
    • Interact with the telnet terminal (not implemented).

Planned Features

  • Telemetry Retrieval: Get the latest telemetry data (not implemented).
  • Command Sending: While planned, this feature is complex and not currently available.

Remote Desktop

The remote desktop functionality uses a web version of VNC, allowing you to connect to the pre-configured machine. This setup aims to resolve the issues we experienced in the past (hopefully!).
